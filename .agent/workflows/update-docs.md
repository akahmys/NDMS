# /update-docs ワークフロー

このワークフローは、プロジェクトの `index.html` および `project.json` の最新の状態を解析し、`specs.md` を自動的に更新するためのトリガーとして機能します。

## 実行手順

1. **変更箇所の抽出**:
   - `index.html` の直近の修正履歴、または `git diff` 等で差分を確認し、新規メソッドや UI 要素を特定します。

2. **スキルの適用**:
   - `.gemini/skills/update-specs/SKILL.md` に定義された手順に従い、`specs.md` の更新内容を構成します。

3. **ドキュメントの同期**:
   - `specs.md` を書き換え、実装と文書の不整合を解消します。

4. **完了報告**:
   - 変更内容を要約し、ユーザーに報告します。
   - 必要に応じて `rules.md` のバージョン更新（vX.X -> vX.Y）を提案します。
