# /push ワークフロー

このワークフローは、実装の完了や進捗の区切りとして、変更をリモートの GitHub リポジトリ (`origin main`) に反映させるための自動化ステップを提供します。

## 実行手順

1. **変更内容の要約**:
   - `git status` を実行して差分を確認します。
   - コミット内容を簡潔な日本語または英語で要約します。

// turbo
2. **プッシュスキル（push-to-github）の実行**:
   - `.gemini/skills/push-to-github/SKILL.md` の手順に従い、以下の操作を実行します。
   - `git add .`
   - `git commit -m "[NDMS] {要約された内容}"`
   - `git push origin main`

3. **デプロイの通知**:
   - プッシュ完了後、GitHub Pages のデプロイが開始されたことを報告します。
   - 公開 URL (`https://akahmys.github.io/NDMS/`) を再提示し、進捗を確定させます。

## トリガーの基準
- `task.md` の主要フェーズ（環境構築、コアロジック等）が完了したとき。
- ユーザーから「プッシュして」または「反映して」という依頼を受けたとき。
- エラー修正が確認され、安定版として保存したいとき。
