# Document Management System (DMS) 詳細仕様書 (GEMINI.md v3.4 準拠)

## 1. システム概要
DMSは、ローカルPC内のフォルダとシームレスに連携し、ドキュメント（主にPDFファイル）のメタデータ管理、プレビュー表示、ソート、分類、および最終的なPDF成果物統合を行うためのブラウザ完結型Webアプリケーションです。

### 1.1. システム要件
* **実行環境**: モダンブラウザ（Google Chrome, Microsoft Edge など）
  * ※ローカルファイルシステムに直接アクセスするため、**File System Access API** の対応が必須です。
* **アーキテクチャ**: Next.js (App Router) + TypeScript。静的 export によるポータブルな配布を前提。

---

## 2. アーキテクチャとテクノロジースタック
* **フレームワーク**: Next.js 16.2.1, React 19
* **言語**: TypeScript
* **スタイリング**: Tailwind CSS v4, CSS カスタムプロパティによるテーマ管理
* **アニメーション**: **Framer Motion** (ツリーの開閉、ドラッグ時、楽観的更新の遷移に使用)
* **アイコン**: Lucide React
* **主要ライブラリ**:
  * **@dnd-kit/core / sortable**: ドラッグ＆ドロップによる要素移動・順序管理
  * **pdf-lib**: PDFファイルの結合、テキスト描画（目次作成）
  * **pdfjs-dist**: PDF解析・サムネイル生成用のレンダリング

### 2.1. データ管理 (Source of Truth)
* プロジェクトフォルダ直下の `project.json` を唯一の正解（Source of Truth）とします。
* 物理ファイル名には連番を含めず、順序や表示名はメタデータ内で管理されます。

### 2.2. 文字コードと正規化 (NFC) 【最優先】
* macOS (NFD) と Windows (NFC) のファイル名不整合を根絶するため、すべてのパス操作において `Utils.norm()` による **NFC 正規化を強制適用**しています（憲法 第3.3条）。

## 3. 主要モジュールと設計

### 3.1. projectService (Business Logic)
* **ファイル同期**: メタデータの更新に伴う物理ファイルのリネーム・移動（`renameCategory`, `moveDocument` 等）を、再帰的な権限確認を行いつつ実行します。
* **バリデーション**: 重複ファイル名、特殊文字の制限。

### 3.2. useMetadata (State Management)
* **Optimistic UI (楽観的更新)**: 物理ファイル操作の完了を待たずに UI を書き換えることで、ドラッグ＆ドロップ時の「ラグ感」を排除しています。
* **Syncing**: 物理操作が失敗した場合には自動的に元に戻すロールバック機能を備えています。

### 3.3. useSelection (Interactions)
* **マルチセレクト**: 単一選択、範囲選択（Shift）、追加選択（Ctrl/Cmd）をサポート。
* **Marquee Selection**: 空白地帯からの Ctrl+ドラッグによる矩形選択機能。

---

## 4. ユーザーインターフェース

### 4.1. Navigator (treeView)

* **高度な操作性**: 
    - **ダブルクリックリネーム**: インラインでのフォルダ名・ファイル名変更。
    - **スプリングロード**: ドラッグ中にフォルダの上でホバーすると、自動的にそのフォルダが展開される機能。
    - **キーボードナビゲーション**: 矢印キー（↑↓←→）による高速なノード移動と展開。
* **3階層構造**: 視認性と管理のしやすさを維持するため、カテゴリは最大3階層までのネストを推奨。

### 4.2. Document Library (thumbnailGrid)

* **独立した作業領域**: 中央ペインは左ペイン（Navigator）から形式的に独立しており、グリッド内での選択や並び替えはツリー表示に影響を与えません。
* **サムネイル表示**: `pdfjs-dist` による 1ページ目の自動生成とプレビュー機能。
* **ローカル・ドラッグ＆ドロップ**: グリッド内での一時的な並び順変更が可能（結合順序の最終確認用）。
* **洗練されたツールバー**: 「**移動**」と「**結合**」に特化した日本語インターフェースを提供。

### 4.3. Context Panel (infoPanel)

* **PDF プレビュー**: ブラウザ標準の PDF ビューアまたは `pdf.js` による表示。
* **メタデータ編集**: 書類タイトル、備考などの編集。

## 5. ユーザーフロー

### 5.1. ドキュメントの分類 (DnD)
1. 中央ペインのドキュメントを選択。
2. 左ペインのターゲットカテゴリへドラッグ。
3. 楽観的更新により、UI 上で即座に移動が完了。
4. ファイルシステム API が背後で物理ディレクトリへの移動を実行。

### 5.2. キーボード操作フロー
1. ツリービュー内のアイテムをクリック（フォーカス取得）。
2. ↑↓ キーで移動。
3. Space / Enter キーで選択。
4. → キーでフォルダ展開。

### 5.3. 範囲選択フロー
1. `Ctrl` (または `Cmd`) キーを押下。
2. ツリービューの空白地帯からドラッグを開始。
3. 描画される青い矩形の中に含まれる全ドキュメントを一括選択。

## 6. プロジェクト構成 (Directory Structure)

プロジェクトのソースコードは、憲法 第7.2条（ファイル名命名規則）に基づき、すべて **camelCase** で命名・整理されています。

### 6.1. src ディレクトリ

* **app/**: Next.js App Router のページコンポーネント、レイアウト、グローバル CSS。
* **components/**: UI コンポーネント群。
  * **tree/**: サイドバーのツリービュー（`treeView`）関連。
  * **document/**: ドキュメントのグリッド表示（`thumbnailGrid`）関連。
  * **info/**: メタデータ表示・編集パネル（`infoPanel`）および各タブ。
  * **ui/**: 共通 UI 要素（`errorToast`, `button` 等）。
* **hooks/**: カスタムフック（`useMetadata`, `useTreeKeyboardNav` 等）。
* **services/**: ビジネスロジックの抽象化（`projectService`, `fileSystemService` 等）。
* **contexts/**: グローバルステート管理（`projectContext`）。
* **lib/**: 共通ユーティリティ、定数。
* **types/**: TypeScript の型定義。

### 6.2. その他

* **docs/legacy/**: 過去の参照用ファイルのアーカイブ。
* **public/**: 静的アセット。
