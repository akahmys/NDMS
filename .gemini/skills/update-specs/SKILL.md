# update-specs スキル

このスキルは、Document Management System (DMS) の開発において、機能追加や仕様変更が行われた後に `specs.md` を最新状態に保つためのガイドラインを提供します。

## 更新のタイミング
- 新しい主要クラス・メソッドが追加されたとき
- メタデータ (`project.json`) の構造が変更されたとき
- UI の主要な構成やデザイン方針が変更されたとき
- 外部ライブラリの追加・バージョン変更が行われたとき

## 手順

### 1. 実装内容の把握
まず、直近の変更（`index.html` の編集履歴など）を確認し、仕様書に反映すべき項目をリストアップします。
- **データ構造**: フィールドの追加・削除
- **ロジック**: 物理ファイル操作、同期処理の変更
- **UI**: 新しい入力項目、ビューモードの追加

### 2. specs.md の該当セクションの特定
`specs.md` は以下の構成になっています。変更内容に最も近いセクションを特定してください。
- `2. アーキテクチャ`: ライブラリ、データ管理方針
- `3. 主要モジュール`: クラスごとの詳細（FileSystemService, ProjectService等）
- `4. ユーザーインターフェース`: 各ペインの役割
- `5. ユーザーフロー`: 操作手順の変更

### 3. 用語の一貫性の維持
既存の用語（例：「カテゴリ」「書類番号」「NFC正規化」など）と一貫性のある言葉遣いで記述してください。

### 4. 特記事項・制約の追記
新しく導入された技術的制約や、特定のブラウザ向けの回避策（フォールバック等）がある場合は、必ず追記してください。

## 注意事項
- **NFC正規化の言及**: ファイルパス操作に関する変更の場合は、必ず `Utils.norm()` による正規化に触れること。
- **シングルファイルの維持**: 構成変更があったとしても、`index.html` 単一ファイルであるという前提を崩さないこと。
